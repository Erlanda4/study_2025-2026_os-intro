# Теоретическое введение
#(Password Store): Простой и удобный менеджер паролей, следующий философии Unix. Каждый пароль хранится в отдельном зашифрованном файле (GPG), организованном в понятную файловую структуру. Поддерживает версионирование через Git 

GPG (GNU Privacy Guard): Свободная реализация протокола OpenPGP. Используется для шифрования и подписывания данных. В контексте pass — для шифрования файлов с паролями. GPG использует пару ключей: приватный (секретный) для расшифровки и публичный для шифрования 

Git: Система контроля версий, используемая pass для синхронизации хранилища между устройствами и отслеживания истории изменений. pass автоматически создает коммиты при изменении паролей
